# NDX Intelligence

NDX Intelligence is an independent, open research index for Data Science and Artificial Intelligence.

It is designed as a public research memory rather than a portfolio. The goal is to document how problems are framed, how systems are built, how experiments evolve, and how understanding changes over time.

The emphasis is on traceability, not presentation.

---

## What this repository represents

This repository contains research work that prioritizes:

- clear problem framing
- explicit assumptions
- system-level thinking
- experimentation under real constraints
- honest failure analysis
- reasoning that compounds over time

Work here may be incomplete, paused, revised, or abandoned. That is intentional. The repository reflects the process of building understanding, not just final outcomes.

---

## What this is not

NDX Intelligence is **not**:

- a tutorial series
- a step-by-step learning path
- a course or certification resource
- a startup codebase
- a consultancy showcase
- a highlight reel of finished projects

Readers looking for simplified explanations or guided instruction should not expect that here.

---

## Repository structure

This repository is organized to preserve reasoning and context:

- **START-HERE.md**  
  How to read this repository and how the structure is intended to be used.

- **index/**  
  Stable summaries of experiments, capturing assumptions, failures, and outcomes.

- **experiments/**  
  Active or archived experiment workspaces.

- **notebooks/**  
  Exploratory analysis, modeling, and evaluation work.

- **notes/**  
  Explicit records of assumptions, failures, and unresolved questions.

- **systems/**  
  Documentation of pipelines, constraints, and system-level decisions.

- **guides/**  
  Meta-level reflections on methodology, framing, and how work is approached.

---

## Philosophy

NDX Intelligence operates on a few core principles:

- Knowledge should be open.
- Disciplines should not be gated by background.
- Learning should compound rather than reset.
- Failures are first-class artifacts.
- Systems matter more than isolated models.
- Reasoning is more important than performance alone.

These principles shape what work is included and how it is documented.
